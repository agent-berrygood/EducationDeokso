# 대규모 개편 구현 계획서 (Major Overhaul Implementation Plan)

본 계획서는 학부모 신청 절차 간소화 및 부모 중심의 데이터 구조, 통합 관리자 운영 체계를 위한 대규모 개편안을 기획 및 구현하기 위한 명세서입니다.

---

## 1. 주요 요구사항 및 아키텍처 설계

### 1.1. 메인 랜딩 페이지 간소화 (`app/page.tsx`)
- 기존의 부서 카드와 설명 링크를 모두 숨깁니다.
- 화면 중앙에 세련된 그라데이션 타이틀과 함께 **`2026 지금세대교육부 여름 캠프 신청하기`**라는 단 하나의 메인 버튼만 남겨 가독성을 극대화합니다.
- 해당 버튼 클릭 시 바로 풀페이지 신청 라우트인 `/apply`로 연결되도록 설계합니다.

### 1.2. 풀페이지 신청 라우트 구현 (`app/apply/page.tsx`)
- 기존에 모달창으로 협소하게 뜨던 `ApplicationForm` 컴포넌트를 걷어내고, 독립된 풀페이지 라우터인 `/apply` 경로를 신설합니다.
- 모바일 기기에서의 입력 피로도를 대폭 낮추고 이탈률을 방지하기 위해 가독성이 높은 널찍한 반응형 폼 인터페이스를 제공합니다.

### 1.3. 신청 프로세스 다단계 고도화 (Step-by-Step Wizard)
#### Step 1: 보호자 정보 & 워터풀 선데이 참석자 등록
- 부모 이름, 연락처, 캠프비 입금자명을 입력받습니다.
- **워터풀 선데이 보호자 참석 정보 수집**: 폭포수 주일에 함께 참여하는 보호자(조부모, 부모 등) 정보를 여러 명 자유롭게 동적으로 행을 추가(`+ 보호자 추가`)하여 수집할 수 있는 Dynamic Array Input UI를 탑재합니다. 반드시 1명 이상 기재되어야 다음 단계로 넘어갈 수 있도록 벨리데이션을 진행합니다.

#### Step 2: 자녀 정보 입력 & 실시간 부서 홍보 콘텐츠 매핑
- 자녀의 이름, 생년월일, 성별을 기입하면 한국 나이를 기반으로 대부서가 자동 판단되거나 혹은 학부모가 직접 대부서(나우킨더, 나우키즈, 나우틴즈)를 선택할 수 있게 합니다.
- **실시간 부서 매핑 비주얼 보드**: 자녀 등록 카드 내에서 부서가 매핑되거나 선택되는 즉시, 해당 부서의 **홍보 포스터 이미지(Base64), 올해의 주제 성구, 행사 타이틀, 테마 슬로건**을 카드 내부에 세련되게 바인딩하여 실시간으로 렌더링합니다.
- 자녀가 두 명 이상일 경우, `자녀 추가`를 누르고 다음 자녀의 부서를 선택하면 그에 맞는 부서 홍보 포스터와 상세 정보가 각각 독립된 카드에 실시간 매핑됩니다.
- 세부부서 선택란(통합미취학부, 영유아부, 유치부, 통합아동부, 유년부, 소년부, 중등부, 고등부)과 참석일정 옵션을 자녀별로 입력받습니다.

### 1.4. 부모 중심의 DB 스키마 마이그레이션 (`lib/db.ts` / Postgres)
- 부모를 중심(Parent-Centric)으로 데이터가 유기적으로 엮이도록 관계를 명확히 합니다.
- **마이그레이션 쿼리**:
  ```sql
  -- 1. 폭포수 주일 부모 동반자 정보를 다중 저장하기 위한 JSONB 컬럼 추가
  ALTER TABLE applications ADD COLUMN IF NOT EXISTS waterfall_parents JSONB DEFAULT '[]';
  ```
- **데이터 흐름**: 학부모 1회의 접수로 부모 정보 테이블(`applications`) 아래에 여러 명의 워터풀 보호자 배열 정보(`waterfall_parents`)가 들어가고, 해당 `application_id`를 참조하는 자녀 테이블(`application_children`) 데이터들이 관계형으로 단단히 결합됩니다.

### 1.5. 통합 관리자 페이지 개발 (`app/admin/page.tsx` & `components/AdminDashboard.tsx`)
- 기존에 부서별로 쪼개져 작동하던 관리자 화면을 단 하나의 통합 대시보드인 `/admin` 주소로 단일화합니다.
- **부서 전환 탭**: 상단에 세련된 세 부서 탭(나우킨더, 나우키즈, 나우틴즈)을 배치하여 클릭 시 해당하는 부서 자녀들의 데이터만 동적으로 가져옵니다.
- **세부부서 2차 탭 필터링**: 선택된 대부서 내부에서 각 부서에 속한 세부부서별(예: 나우키즈 선택 시 -> 전체, 통합아동부, 유년부, 소년부) 탭을 2차적으로 탑재하여 교사들이 해당 세부 부서의 학생들만 한눈에 필터링해 볼 수 있게 완벽하게 조율합니다.

## 2. 발생 가능 오류 사전 진단 및 예외 조치

> [!IMPORTANT]
> - **데이터 마이그레이션**: 기존의 데이터 유실을 완벽히 차단하기 위해 `ALTER TABLE` 구문을 수행하되, `DEFAULT '[]'`와 `IF NOT EXISTS` 가드를 촘촘하게 둘러 Null 에러를 철저히 차단합니다.
> - **Base64 이미지 처리**: 자녀의 부서 선택 시 해당 부서 정보가 실시간 렌더링될 때, 용량이 큰 Base64 데이터 로드 속도로 인한 로딩 딜레이를 유발하지 않도록 이미지 스켈레톤 홀더를 깔끔하게 적용합니다.
> - **필터 성능 인덱싱**: `/admin`으로 통합 쿼리가 날아갈 때 다량의 JOIN 연산 속도가 저하되지 않도록, 자녀 소속 부서(`department`, `sub_department`)에 적절한 복합 인덱스가 인지되도록 관리합니다.

## 3. 검증 계획

- `npm run build`를 동작시켜 Next.js 16 빌드가 정상적으로 컴파일 완료되는지 통과 여부를 정밀 분석합니다.

### 3.2. 수동 검증
- `/apply` 풀페이지에서 다중 자녀 및 폭포수 보호자 동적 배열 추가 작동 테스트.
- `/admin` 단일 통합 패널에서 탭 전환 시 DB 데이터 동적 로드 및 세부부서 실시간 필터 기능 동작 테스트.
